# Lume

> 🌐 **Languages:** **English** · [简体中文](README.zh-CN.md)

A local, offline PDF toolkit packaged as a native macOS app. Every document stays on your device — nothing is uploaded, synced, or phoned home.

## Features

- **Reader** — view, search, highlight, underline, and annotate; bookmarks and thumbnail navigation
- **Merge** — combine multiple PDFs into one
- **Organize pages** — drag to reorder, delete, and rotate pages
- **Convert** — PDF ↔ images (PNG / JPG)
- **Sign** — draw a signature and embed it into a document
- **Watermark** — add text watermarks
- **Recent documents** — history kept locally in IndexedDB for quick reopening

## Install (end users)

Download `Lume-x.y.z-arm64.dmg` (Apple Silicon) or `-x64.dmg` (Intel) from the [Releases page](https://github.com/xiangjie-zhu/Lume/releases). Double-click the DMG and drag Lume into Applications.

The first time you open it, macOS will warn that the developer can't be verified — this is expected (the DMG isn't notarized, since this is a personal project without an Apple Developer account). Right-click the app icon and choose **Open**, or run:

```bash
xattr -cr /Applications/Lume.app
```

## Run from source (developers)

**Requirements:** Node.js 20+ and macOS.

```bash
git clone https://github.com/xiangjie-zhu/Lume.git
cd Lume
npm install

# Browser preview (Vite dev server)
npm run dev

# Desktop mode (hot reload + DevTools)
npm run electron:dev
```

## Build your own DMG

```bash
npm run dist:mac
```

Output lands in `dist-electron/Lume-<version>-arm64.dmg`. On Apple Silicon, `electron-builder` defaults to arm64 only — to also produce an Intel build:

```bash
./node_modules/.bin/electron-builder --mac dmg --x64
# or a universal binary
./node_modules/.bin/electron-builder --mac dmg --universal
```

## Automated test

An end-to-end smoke test launches Electron, injects a test PDF, verifies pdfjs renders a canvas, and confirms the history record persists:

```bash
npm run build
./node_modules/.bin/electron scripts/test-pdf.cjs
```

Expected output:

```
✓ PDF rendered successfully (canvas 150x150)
✓ History recorded and visible on Home Dashboard
```

## Customize the icon

The app icon is generated by `scripts/build-icon.cjs` — it renders a snippet of HTML via Electron's offscreen capture, then packages the PNG variants into an `.icns` bundle with `iconutil`. After changing the design:

```bash
npm run icon
```

Outputs: `build/icon.png`, `build/icon.icns`.

## Tech stack

- **UI:** React 19 + Vite + Tailwind v4 + Motion
- **PDF:** [`pdfjs-dist`](https://github.com/mozilla/pdf.js) (rendering) + [`pdf-lib`](https://pdf-lib.js.org/) (editing)
- **Desktop shell:** Electron 33 + electron-builder
- **Storage:** localforage (IndexedDB wrapper)

Assets are served through a custom `app://` protocol (instead of `file://`) to give the pdfjs Worker a proper origin — workers can't instantiate reliably under `file://`.

## Project layout

```
src/
  App.tsx            # top-level layout, tab management
  main.tsx           # React entry + pdfjs worker setup
  components/        # shared components (HomeDashboard)
  tools/             # tool implementations (ReaderTool, ConvertTool, ...)
  lib/
    storage.ts       # recent documents (localforage + event bus)
    utils.ts         # download + formatting helpers
electron/
  main.cjs           # Electron main process; registers app:// protocol
scripts/
  build-icon.cjs     # icon generator
  test-pdf.cjs       # end-to-end smoke test
build/
  icon.icns          # app icon (committed; used at package time)
  icon.png
```

## Known gotchas

- **"Failed to load PDF"** — almost always a version mismatch between `pdfjs-dist` and the copy bundled inside `react-pdf`. The `pdfjs-dist` version in `package.json` must match `react-pdf/node_modules/pdfjs-dist` exactly (currently `5.4.296`).
- **macOS says the DMG is "damaged"** — it's just the quarantine flag from downloading an unnotarized build. Clear it with `xattr -cr /Applications/Lume.app`.
- **New icon doesn't show up after rebuilding the DMG** — macOS LaunchServices caches icons:
  ```bash
  /System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -kill -r -domain local -domain system -domain user
  killall Dock Finder
  ```
